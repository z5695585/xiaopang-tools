API 路由 & AI 总结

RESTful API + DeepSeek 周报/月报生成

API 路由设计

方法 路径 说明
GET/api/todos获取待办列表（支持 ?tag= & ?status=）
POST/api/todos创建待办
PUT/api/todos/:id更新待办（标题、完成状态等）
DELETE/api/todos/:id删除待办
GET/api/todos/completed?period=week|month按周/月查已完成（含子待办）
POST/api/summary/generate生成 AI 周报/月报
GET/api/tags获取所有标签
POST/api/tags创建标签

AI 总结流程

1. 用户点击
"生成本周周报"
→
2. 后端查询
本周完成的待办+子待办
→
3. 组装 Prompt
按模板格式发给 DeepSeek
→
4. 返回总结
展示在页面上，可复制

周报/月报格式（可配置）

## 周报（{日期范围}）

### 本周完成
- [标签:工作] 完成了XX功能的接口开发
  - 子任务：数据库表设计
  - 子任务：API 调试通过
- [标签:学习] 阅读了DeepSeek API文档
  - 子任务：跑通Quick Start示例

### 本周总结
{AI 根据完成内容自动生成的总结段落}

### 下周计划（可选）
{AI 根据未完成待办推测的下周计划}
